# Directory structure

This document enumerates the directory layout of `apps/editor/src/` with a short description of every significant folder. For the layer model and runtime behavior see [architecture.md](./architecture.md). For package-internal structure see [packages.md](./packages.md).

## Top-level

```
apps/editor/src/
├── main.tsx              # Mounts <App /> into #root
├── main.css              # Imports Tailwind + design tokens
├── routeTree.gen.ts      # Auto-generated by @tanstack/router-vite-plugin — do not edit
├── vite-env.d.ts         # Vite env typings
├── assets/               # Static assets (images, fonts)
├── storybook/            # Storybook decorators and helpers scoped to apps/editor
├── core/                 # Domain modules + app shell (see below)
├── components/           # Reusable UI components (see below)
├── views/                # Page-level compositions consumed only by routes
└── routes/               # TanStack file-based routes
```

## `core/` — domain modules and app shell

```
core/
├── App.tsx               # Root: <AppProviders><AppTheme><AppRoutes /></AppTheme></AppProviders>
├── AppProviders.tsx      # Provider stack (see architecture.md)
├── AppLocales.tsx        # React Intl setup — loads messages for the active locale
├── AppTheme.tsx          # Applies theme class based on themeModeAtom
│
├── auth/                 # Authentication domain
│   ├── AuthAtoms.ts           # Jotai atoms — auth state
│   ├── AuthInitializer.tsx    # Bootstraps auth on mount
│   ├── AuthState.ts           # Domain type: auth state shape
│   ├── AuthStatus.ts          # Domain type: status enum
│   └── UseAuth.ts             # Hook exposing { login, logout, ... }
│
├── config/               # App-level config constants / lookups
│   └── UseAppInfo.ts          # App name, version, build info
│
├── pwa/                  # Progressive Web App lifecycle
│   ├── PwaLifecycle.tsx       # Provider that wires update/offline events
│   ├── PwaLifecycleConstants.ts
│   └── UsePwaLifecycle.ts
│
├── routes/               # Router setup (not routes themselves — those live at src/routes/)
│   ├── AppRoutes.tsx          # createRouter + <RouterProvider>
│   └── logic/
│       ├── RouteError.tsx     # Error UI shown when a route throws
│       └── RouteLoading.tsx   # Suspense fallback for route transitions
│
├── session/              # Authenticated session content (name, email, etc.)
│   ├── AppSessionContent.ts
│   └── UseAppSessionContent.ts
│
├── settings/             # Settings modal state
│   ├── SettingsAtoms.ts       # Open/close + active section atoms
│   ├── SettingsSection.ts     # Section enum
│   └── UseSettingsModal.ts    # Hook exposing { isOpen, open, close, section }
│
├── social-links/         # Links to external socials
│   ├── AppSocialLinks.ts
│   └── UseAppSocialLinks.ts
│
└── theme/                # Theme mode (light/dark/system)
    ├── ThemeAtoms.ts          # themeModeAtom + atomEffect persistence
    ├── ThemeMode.ts           # Domain type
    └── UseTheme.ts            # Hook exposing { mode, setMode }
```

**Conventions for a `core/` feature folder:**

- `XxxAtoms.ts` for Jotai state (optional).
- `UseXxx.ts` for the primary hook exposing the feature.
- PascalCase files for types, constants, and components.
- Sub-components stay here **only** if they are inseparable from the domain (e.g. `AuthInitializer.tsx`, `PwaLifecycle.tsx`). Pure UI belongs in `components/`.

## `components/` — reusable UI

```
components/
├── actions/
│   ├── command-palette/        # Cmd/Ctrl+K overlay (Controller pattern)
│   ├── quick-menu/             # Dropdown of session/appearance actions
│   └── theme-selector/         # Radio-style theme switcher
│
├── display/
│   └── grid-item/              # Small card for list/grid contexts
│
├── feedback/
│   ├── about-details/          # About modal content
│   ├── pwa-offline-dialog/     # Shown while offline
│   ├── pwa-update-dialog/      # Shown when a new SW version is ready
│   └── settings-modal/         # Settings overlay (Controller pattern)
│
├── forms/
│   ├── forgot-password/
│   ├── login/
│   ├── shared/                 # Form fragments shared between forms
│   └── sign-up/
│
├── input/
│   └── input-field/            # Tanstack-Form-integrated text field
│
└── navigation/
    └── navbar-content/         # User menu + search used inside NavbarLayout
```

**Conventions for a component folder:**

- `<ComponentName>.tsx` — the component itself.
- `<ComponentName>.stories.tsx` — mandatory story file (see [naming.md](./naming.md#story-titles)).
- `<ComponentName>Controller.tsx` — mandatory when using the Controller pattern. See [patterns.md#controller-pattern](./patterns.md#controller-pattern).
- `Use<ComponentName>Controller.ts(x)` — the controller's hook.
- Private sub-components and types stay in the same folder unless they are used from more than one component.

## `views/` — page-level compositions

```
views/
├── dashboard/
│   └── DashboardView.tsx
├── forgot-password/
│   └── ForgotPasswordView.tsx
├── home/
│   └── HomeView.tsx
├── login/
│   └── LoginView.tsx
└── sign-up/
    └── SignUpView.tsx
```

Each view is a **pure presentational component** that receives every value it needs via props. Views never call hooks from `core/` or `@package/api` directly — that is the route hook's job.

A view folder contains:

- `<Name>View.tsx`
- `<Name>View.stories.tsx`
- Private sub-components used only by this view.

## `routes/` — TanStack file-based routing

```
routes/
├── __root/
│   └── route.tsx                           # Root layout (<Outlet />)
├── $/
│   └── route.tsx                           # 404 catch-all
│
├── _public/                                # Layout: unauthenticated users only
│   ├── route.tsx                           # Guard + shared public chrome
│   ├── forgot-password/
│   │   ├── route.tsx
│   │   └── -UseForgotPasswordRoute.ts
│   ├── login/
│   │   ├── route.tsx
│   │   └── -UseLoginRoute.ts
│   └── sign-up/
│       ├── route.tsx
│       └── -UseSignUpRoute.ts
│
└── _authenticated/                         # Layout: authenticated users only
    ├── route.tsx                           # Guard + navbar layout
    ├── -UseAuthenticatedRoute.ts
    ├── index/
    │   └── route.tsx                       # "/" (home)
    └── dashboard/
        ├── route.tsx
        └── -DashboardLoader.ts             # React Query prefetch loader
```

**Conventions for a route folder:**

- `route.tsx` is the only file TanStack Router reads as a route. It exports `Route = createFileRoute(...)({ component, loader, ... })`.
- Files prefixed with `-` (e.g. `-UseLoginRoute.ts`, `-DashboardLoader.ts`) are ignored by TanStack Router and are **private to the route**.
- The route component is a thin wrapper: it calls the route hook and renders the view.
- Layout routes (`_authenticated`, `_public`) apply guards and shared UI via `<Outlet />`.

Example `route.tsx`:

```tsx
import type { ReactElement } from "react";
import { createFileRoute } from "@tanstack/react-router";
import { useLoginRoute } from "./-UseLoginRoute";
import { useDocumentTitle } from "~/core/routes/logic/UseDocumentTitle";
import { LoginView } from "~/views/login/LoginView";

export const Route = createFileRoute("/_public/login")({ component: LoginPageRoute });

function LoginPageRoute(): ReactElement {
  useDocumentTitle("Login");
  const props = useLoginRoute();
  return <LoginView {...props} />;
}
```

## Import aliases

The `~/` alias resolves to `apps/editor/src/`. Prefer it for cross-layer imports:

```ts
import { useAuth } from "~/core/auth/UseAuth";
import { LoginView } from "~/views/login/LoginView";
import { InputField } from "~/components/input/input-field/InputField";
```

Relative imports (`./`, `../`) are reserved for files within the same folder.
